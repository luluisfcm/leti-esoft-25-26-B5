# US10 - Specify New Vaccine Type

## 1. User Story

**Como um:** Administrador DGS
**Quero:** Especificar um novo tipo de vacina
**Para que:** O sistema possa utilizar esse tipo para registar vacinas específicas e gerir o processo de vacinação.

## 2. Descrição

Esta funcionalidade permite ao Administrador DGS gerir os dados centrais da aplicação. O administrador pode definir um novo tipo de vacina que o sistema irá utilizar.

De acordo com os requisitos, um tipo de vacina deve incluir:
* [cite_start]A doença (disease) que combate [cite: 64]
* [cite_start]Uma breve descrição (short description) [cite: 64]

## 3. Pré-condições

1.  [cite_start]O Administrador DGS deve estar autenticado no sistema (pode ser simulado)[cite: 73].

## 4. Pós-condições

1.  Um novo tipo de vacina é criado e guardado permanentemente no sistema.
2.  O sistema regista a operação (sucesso ou falha).

## 5. Critérios de Aceitação (Acceptance Criteria)

* **AC1 (Dados Obrigatórios):** O tipo de vacina deve incluir a doença alvo e uma breve descrição.
* **AC2 (Doença Única):** O nome da `disease` (doença) deve ser único. O sistema não deve permitir o registo de um novo tipo de vacina com uma doença que já exista.
* **AC3 (Feedback Sucesso):** O sistema deve confirmar ao administrador que o novo tipo de vacina foi registado com sucesso.
* **AC4 (Feedback Falha):** Se o registo falhar (ex: doença duplicada), o sistema deve informar o administrador da causa do erro.
